I've updated `database.py` and `PERFORMANCE_FIX_LOG.md` to add several missing database indexes for performance tuning, specifically on the `inventory`, `players`, `pvp_logs`, `content`, `diary` and other high-utilization tables that previously saw high sequential scan counts.

The indexes are added using the Supabase API via MCP for existing tables and then a new function `fix_indexes` is added to `database.py` that gets called during `init_db()` to create them on fresh deployments. Tests like `test_pvp_service`, `test_raid_cost`, and `test_raid_loot` all still pass.
