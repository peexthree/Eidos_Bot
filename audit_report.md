# Аудит проекта EIDOS: Chronicles (v8.5)

## 🔍 1. Анализ текущего состояния (Слабые места и недочёты)

**Бэкенд (API & бот):**
1. **Массовые заглушки в API:** Эндпоинты `/api/action/synchron`, `/api/action/signal`, `/api/action/craft` не имеют связи с базой данных и игровой логикой. Они отдают замоканные JSON-ответы.
2. **Опасный кэш инвентаря:** Использование глобальной переменной `global _inventory_cache` в `bot.py` для логики экипировки (`equip` / `unequip`) приведет к рассинхронизации данных у пользователей при многопоточности.
3. **Небезопасный крафтинг:** Эндпоинт крафта просит "серверную синхронизацию" (возвращает 501).
4. **Синхронизация с TWA (sendBeacon):** Роут `/api/twa/sync` работает, но сброс кэша Redis может отставать, из-за чего бот не сразу видит изменения.

**Фронтенд (React / TWA):**
1. **Интерфейс Hub.jsx — заглушка:** Почти все кнопки на главном экране привязаны к `handleAction`, который вызывает `twa.showPopup("SYSTEM MODULE OFFLINE")`. Нет реальной навигации.
2. **Проблема стейта Инвентаря (Zustand):** Сильный рассинхрон нейминга слотов (на фронте `armor`, `chip`, с бэкенда `body`, `software`). Предметы "зависают", оптимистичные апдейты ломают UI.
3. **Обработка ошибок в ItemModal:** Если запрос во время HoldToEquip упадет, нет локального rollback-состояния.
4. **Абсолютное позиционирование UI:** Слишком много жестких процентов (`top: 18%`) в `Hub.jsx`. На нестандартных экранах кнопки будут съезжать.

## 🛠 3. Список проблем и их архитектурные решения (без кода)

1. **"Фантомные" шмотки (Рассинхрон слотов):** Создать единый словарь (`apiSlotMap`) на фронте, который строго переводит фронт-нейминг в бэк-нейминг. При ошибке от бэка — вызывать функцию отмены локального стейта (revertOptimistic).
2. **Рассинхрон бота и TWA:** В роуте `/api/twa/sync` принудительно делать удаление кэша (delete_user_cache), чтобы следующий запрос в Telegram дергал свежие данные из PostgreSQL.
3. **Заглушки API (501 Not Implemented):** Внутри API роутов вызывать уже существующие методы обновления статистики (например, `db.update_user_stats`) внутри транзакции.
4. **Заглушки Hub.jsx:** В `App.jsx` расширить `switch(currentView)`. В `Hub.jsx` при нажатии вызывать `setView`, скрывая видео Хаба и показывая нужный компонент.
5. **Падение производительности UI:** Увеличить `staleTime` для профиля в React Query, мутации должны локально обновлять кэш без повторного тяжелого запроса профиля.

## 💡 4. Топ-10 креативных решений для повышения функциональности

1. **PWA Service Worker:** Закэшируйте тяжелые фоновые видео и картинки через Service Worker. Апп будет открываться мгновенно.
2. **Свайп-крафт (Smart Batching):** Позвольте выделять предметы долгим тапом и отправлять пакетный запрос на бэкенд (разгрузит БД).
3. **Глитч-индикатор пинга:** Если запрос идет дольше 800мс, накладывайте легкий CSS-фильтр хроматической аберрации (задержка = "сюжетный сбой связи").
4. **Акустический Haptic:** При удержании кнопки (HoldToEquip) параллельно с вибрацией смартфона запускайте нарастающий звук "заряда энергии".
5. **Терминальные уведомления:** Замените обычные alert на лог действий зеленым моноширинным шрифтом сверху экрана, как в консоли Linux.
6. **"Теневой" Инвентарь (Offline-first):** Сохраняйте стейт в `IndexedDB`. Позвольте игроку переодеваться даже без сети, отправляя очередь действий при появлении связи.
7. **Dynamic Hub Lighting:** Меняйте CSS-фильтры Хаба от состояния. Здоровье < 20%? Красная пульсирующая виньетка.
8. **Шейдерная магия редкости:** Реликвии (красные) должны иметь блики, реагирующие на гироскоп телефона (`DeviceOrientationEvent`).
9. **Скелетоны "Матрицы":** При загрузке показывайте силуэты карточек, по которым бегут зеленые символы, расшифровываясь в реальные предметы.
10. **Микро-игры (Hacking):** Если крафтинг или ИИ-профайлинг требуют времени, показывайте простую мини-игру "взлома" (вовремя нажать кнопку), чтобы замаскировать загрузку.
