It seems that almost all glitches (micro-glitches) use `bot.answer_callback_query(call.id, f"🌀 {glitch['message']}", show_alert=True)`.
In some places like `gameplay.py` (get_protocol, get_signal), it does not use `answer_callback_query` for the glitch message, but appends it to `reward_text = f"\n\n🌀 <b>{glitch['message']}</b>"`.

For `answer_callback_query`, Telegram limits text to 200 chars. If `glitch['message']` is long, it gets truncated and the user doesn't see why the system glitched.

We should wrap the glitch alert in `bot.send_message(uid, f"🌀 <b>СИСТЕМНАЯ АНОМАЛИЯ</b>\n\n{glitch['message']}", parse_mode="HTML")` to make it explicitly visible as a persistent message, in addition to or instead of just a popup that can vanish.

2. `modules/handlers/items.py` (inventory, dismantle)
3. `modules/handlers/gameplay.py` (protocol, signal)
